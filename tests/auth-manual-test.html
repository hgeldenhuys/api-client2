Manual Test Instructions for Auth Input Issue

Issue Description

Error: "Uncaught TypeError: Cannot add property 0, object is not extensible"
Location: AuthenticationEditor.tsx:69 (Array.push)

This error occurs when trying to type in authentication input fields.

Test Steps

Prerequisites: Make sure the dev server is running on http://localhost:5173

Step 1: Open the application at http://localhost:5173
Step 2: Click on "New Request" button to create a new request
Step 3: Click on the "Auth" tab
Step 4: Select "Bearer Token" from the authentication type dropdown
Step 5: Try to type in the Bearer token input field

Expected Results

✓ The input field should accept text without errors
✓ The typed value should appear in the input field
✓ No console errors should occur

What Was Fixed

  • Issue: The auth params array was immutable (from Zustand store)
  • Fix 1: Modified setAuthParam to create a new array instead of mutating
  • Fix 2: Added deep copy logic when editing inherited auth from collection
  • Fix 3: Ensured proper auth object creation when none exists

Additional Test Cases

  1. Test Basic Auth with username and password fields
  2. Test API Key auth with key name and value fields
  3. Test switching between auth types and verify values persist
  4. Test with inherited auth from collection (if available)
